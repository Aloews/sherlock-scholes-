# Инструкция по сбору игроков (для обычного пользователя)

Эта программа сама заходит в интернет, собирает футболистов нужной лиги и
записывает их в базу. Ниже — по шагам, без технических терминов. Делайте
ровно по порядку, первый раз это занимает 15–20 минут.

> Всё, что нужно запускать, — это файлы с названием `sbor_*.bat` и
> `poisk_dublei.bat`. Их запускают **двойным щелчком мыши**. Командная
> строка не нужна.

---

## Шаг 1. Установить Python (один раз)

Python — это программа, которая умеет запускать наш сборщик.

1. Откройте ссылку: **https://www.python.org/downloads/**
2. Нажмите большую жёлтую кнопку **«Download Python»** (версия 3.x).
3. Запустите скачанный файл.
4. **ОЧЕНЬ ВАЖНО:** на первом экране установки внизу поставьте галочку
   **«Add Python to PATH»** (Добавить Python в PATH). Без этой галочки
   ничего не заработает.
5. Нажмите **«Install Now»** и дождитесь конца установки. Готово.

Как проверить, что установилось: запустите любой `sbor_*.bat` двойным
щелчком (шаг 5). Если в окне нет ошибки «python не является внутренней
командой» — значит всё хорошо.

---

## Шаг 2. Получить два «ключа» (один раз)

Ключ — это длинная строка букв и цифр, как пароль. Нужны два ключа.

### Ключ A — API-Football (откуда берутся футболисты)

1. Зайдите на **https://www.api-football.com/** и нажмите **«Sign Up»**
   (Регистрация). Зарегистрируйтесь по электронной почте, подтвердите её
   письмом.
2. Войдите в личный кабинет: **https://dashboard.api-football.com/**
3. На главной странице кабинета (раздел **«Account»** / «My Access») будет
   строка **«API Key»** — длинный набор букв и цифр. Это и есть ключ A.
   Скопируйте его.

> Бесплатный тариф даёт **100 запросов в день**. Этого хватает на одну лигу
> в день. Завтра лимит обнуляется сам.

### Ключ B — Supabase (куда записываются игроки)

1. Зайдите на **https://supabase.com/** и войдите в свой проект (тот, где
   живёт игра).
2. Слева внизу нажмите шестерёнку **«Project Settings»** → пункт **«API»**.
3. Скопируйте два значения с этой страницы:
   - **«Project URL»** — адрес вида `https://xxxxx.supabase.co`
   - **«Project API keys»** → ключ **`service_role`** (секретный). Нажмите
     «Reveal», чтобы увидеть его целиком.

> Ключ `service_role` — секретный, никому не показывайте и не выкладывайте
> в интернет. Он нужен, чтобы программа могла записывать в базу.

## Шаг 3. Создать файл с ключами (.env) — один раз

Программа читает ключи из файла с именем **`.env`** (точка в начале — часть
имени). Его нужно создать один раз.

1. Откройте папку `football_scraper`.
2. Найдите файл **`.env.example`** — это образец. Скопируйте его
   (Ctrl+C, затем Ctrl+V в той же папке). Появится копия
   `.env.example - копия`.
3. Переименуйте копию ровно в **`.env`** (без `.example`, без `.txt`).
   - Если Windows не даёт убрать расширение — включите показ расширений:
     в Проводнике вкладка **«Вид»** → галочка **«Расширения имён файлов»**.
4. Откройте `.env` Блокнотом (правой кнопкой → «Открыть с помощью» →
   «Блокнот») и впишите свои ключи вместо подсказок.

Должно получиться примерно так (подставьте СВОИ значения):

```
FOOTBALL_API_KEY=1a2b3c4d5e6f7g8h9i0jklmnopqrst
SUPABASE_URL=https://abcdefgh.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.очень.длинный.ключ
```

5. Сохраните файл (Ctrl+S) и закройте Блокнот.

> Файл `.env` остаётся только на вашем компьютере и никуда не отправляется.

## Шаг 4. Что делает каждая кнопка (.bat) — простыми словами

| Файл (двойной щелчок)   | Что делает                                                        |
|-------------------------|-------------------------------------------------------------------|
| `sbor_seria_a.bat`      | Собирает игроков **Серии А** (Италия) и записывает их в базу.      |
| `sbor_la_liga.bat`      | Собирает игроков **Ла Лиги** (Испания) и записывает в базу.        |
| `sbor_bundesliga.bat`   | Собирает игроков **Бундеслиги** (Германия) и записывает в базу.    |
| `sbor_ligue1.bat`       | Собирает игроков **Лиги 1** (Франция) и записывает в базу.         |
| `sbor_prosmotry.bat`    | Считает **популярность** игроков по просмотрам Википедии.          |
| `poisk_dublei.bat`      | **Ищет повторы** (дубли) среди карточек. Ничего не удаляет.        |

Подробнее:

- **Сбор лиги** (`sbor_seria_a` и т.п.) — заходит в API-Football, берёт
  заранее выбранный список клубов этой лиги, находит игроков, подбирает
  русские имена и сохраняет их. Это тратит запросы из дневного лимита
  (100 в день). **Одна лига = один запуск в день.**

- **Просмотры** (`sbor_prosmotry`) — берёт уже собранных игроков и узнаёт,
  насколько они известны, по числу просмотров их статей в Википедии. Это
  **не тратит** дневной лимит API-Football (другой, бесплатный источник).

- **Поиск дублей** (`poisk_dublei`) — только смотрит и выводит список
  «карточка A ↔ карточка B», которые похожи и могут быть одним и тем же
  человеком под разным написанием. **Сам ничего не удаляет** — решение за
  вами. Это безопасно запускать в любой момент.

> Все запуски можно повторять: программа не создаёт повторов уже собранных
> игроков, а дописывает только новых.

## Шаг 5. Как запускать (двойной щелчок)

1. Откройте папку `football_scraper`.
2. **Дважды щёлкните** по нужному файлу, например `sbor_seria_a.bat`.
3. Откроется чёрное окно — это нормально. В нём построчно идёт отчёт:
   сколько клубов, сколько игроков, сколько запросов потрачено.
4. Дождитесь надписи в конце. Между клубами программа делает **паузы по
   несколько секунд** — так и должно быть (это вежливость к серверу, не
   зависание).
5. Когда всё закончится, внизу появится **«Press any key to continue…»**
   (Нажмите любую клавишу). Окно не закроется само — спокойно прочитайте
   результат, потом нажмите любую клавишу, чтобы закрыть.

Сначала лучше запустить **`poisk_dublei.bat`** — он безопасный и покажет,
что окно вообще открывается и Python установлен.

## Шаг 6. Частые ошибки и что делать

**«python не является внутренней или внешней командой…»**
Python не установлен или при установке забыли галочку «Add Python to PATH»
(шаг 1, пункт 4). Переустановите Python и обязательно поставьте эту галочку.

**`require_env: FOOTBALL_API_KEY is not set` (или SUPABASE_URL / SUPABASE_KEY)**
Не создан или неправильно заполнен файл `.env` (шаг 3). Проверьте, что файл
называется именно `.env` (а не `.env.txt`), лежит в папке `football_scraper`
и в нём вписаны ваши ключи.

**«Бюджет исчерпан» / `budget exhausted` / в отчёте `100/100`**
Закончился дневной лимит API-Football (100 запросов в сутки). Это не
поломка. Подождите до следующего дня — лимит обнулится сам — и запустите
снова. Уже собранные игроки сохранены, повторный запуск продолжит с того
места. Совет: собирайте **по одной лиге в день**.

**`[warn] клуб 'XXX' не найден в лиге YY, пропущен`**
Это предупреждение, не ошибка. У API-Football клуб называется немного иначе,
чем в нашем списке, поэтому он пропущен. Программа продолжает с остальными
клубами. Если такой клуб важен — сообщите тому, кто настраивает список
клубов (файл `config.json`, раздел `teams_filter`).

**Окно открылось и сразу закрылось**
Запускайте именно `.bat`-файлы из этой папки — в них в конце есть пауза.
Если всё равно закрылось — значит Python не установлен (см. первую ошибку).

**В чёрном окне русские буквы выглядят как «кракозябры»**
На результат это не влияет, данные записываются правильно. Можно не
обращать внимания.

---

### Короткая памятка

1. Один раз: установить Python (с галочкой PATH) → получить 2 ключа →
   создать `.env`.
2. Каждый день: один `sbor_*.bat` на лигу. Потом по желанию
   `sbor_prosmotry.bat` и `poisk_dublei.bat`.
3. Дождаться отчёта, нажать любую клавишу, чтобы закрыть окно.
